# Groove Android — Offline Storage & Metadata Implementation Plan (v2)

> **Revised to fix 10 structural issues** identified in v1.  
> All code samples use the existing Hilt + Retrofit + Room + Media3 stack.

---

## What Changed From v1

| Issue | v1 Problem | v2 Fix |
|-------|-----------|--------|
| 1 | Audio saved to internal app storage | Save to `Music/Groove/` shared storage |
| 2 | TIDAL used during playback | TIDAL only during download; playback uses local file |
| 3 | Local scanned songs used TIDAL | Scanned songs use `MediaMetadataRetriever` only |
| 4 | One monolithic `track_metadata` table | 7 separate, focused tables |
| 5 | Lyrics stored in songs table | Separate `lyrics_cache` table |
| 6 | Full-size artwork cached for every song | Compress → thumbnail; 128px lists, 512px player |
| 7 | Playlists duplicated song data | `playlist_song_map` join table only |
| 8 | No play history | `play_history` + `song_stats` tables from Day 1 |
| 9 | Used `MediaPlayer` | Media3 ExoPlayer with `MediaItem.fromUri()` |
| 10 | Not offline-first | All playback data loadable from local storage only |

## Architecture Overview

### Corrected Data Flow

```
DOWNLOAD FLOW (TIDAL involved):
─────────────────────────────────────────────────────
User taps Download
  → Search/resolve via TIDAL (Smusic backend proxy)
  → Download audio stream
  → Save audio file to: /sdcard/Music/Groove/<Artist>/<Title>.flac
  → Extract TIDAL metadata (title, artist, album, duration, isrc)
  → Compress artwork → thumbnail → save to app cache
  → Write to Room:
      songs            (uri, title, artist, album, duration, isDownloaded=true)
      artwork_cache    (songId, thumbnailPath, listPath)
  → TIDAL is done. Never called again for this song.

PLAYBACK FLOW (offline, no TIDAL):
─────────────────────────────────────────────────────
User taps Play
  → Load song from songs table (uri, title, artist)
  → Load artwork from artwork_cache (thumbnailPath)
  → Load lyrics from lyrics_cache (if available)
  → ExoPlayer: MediaItem.fromUri(song.uri)
  → Record play to play_history + song_stats

SCAN FLOW (no TIDAL):
─────────────────────────────────────────────────────
User picks folder via SAF
  → SafFolderScannerImpl recursively walks DocumentFile tree
  → MediaMetadataRetriever extracts metadata per file
  → Write to songs table (sourceType = "saf")
  → Auto-create playlists from folder names via playlist_song_map
  → TIDAL never involved
```

## Issue 1 Fix — Download Storage Location

Audio files must go to shared storage, not internal app storage.

```kotlin
// DownloadStorageHelper.kt
object DownloadStorageHelper {

    /**
     * Returns the target File for a downloaded track.
     * Path: /sdcard/Music/Groove/<Artist - Title.flac>
     * All downloads sit flat in one folder — no artist subfolders.
     */
    fun getTargetFile(context: Context, artist: String, title: String, ext: String = "flac"): File {
        val safeArtist = artist.replace(Regex("[/\\\\:*?\"<>|]"), "_").take(40)
        val safeTitle  = title.replace(Regex("[/\\\\:*?\"<>|]"), "_").take(80)

        val musicDir  = Environment.getExternalStoragePublicDirectory(Environment.DIRECTORY_MUSIC)
        val grooveDir = File(musicDir, "Groove")
        grooveDir.mkdirs()

        return File(grooveDir, "$safeArtist - $safeTitle.$ext")
    }
}
```

**Required manifest permissions:**
```xml

<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"   android:maxSdkVersion="29" />
```

## Issue 2 & 3 Fix — TIDAL Boundary

### DownloadRepository (TIDAL allowed here)

```kotlin
// Called only during download — TIDAL is isolated to this class
class DownloadRepository @Inject constructor(
    private val songlinkApi: SonglinkApiService,   // Smusic backend proxy
    private val db: GrooveDatabase,
    private val artworkCacheHelper: ArtworkCacheHelper
) {
    suspend fun downloadTrack(track: TidalSearchTrack): Result<Long> {
        return runCatching {
            // 1. Resolve stream URL
            val resolved = songlinkApi.resolve(track.title, track.artist)

            // 2. Write audio to Music/Groove/
            val targetFile = DownloadStorageHelper.getTargetFile(context, track.artist, track.title)
            val proxyUrl = "${BuildConfig.BACKEND_URL}/api/tidal-download/stream?url=${Uri.encode(resolved.streamUrl)}"
            downloadFileToPath(proxyUrl, targetFile)

            // 3. Insert into songs table
            val songEntity = SongEntity(
                title        = resolved.title,
                artist       = resolved.artist,
                album        = resolved.album,
                durationMs   = resolved.durationMs,
                isrc         = track.isrc,
                uri          = Uri.fromFile(targetFile).toString(),
                sourceType   = "tidal_download",
                isDownloaded = true,
                downloadedAt = System.currentTimeMillis(),
                artworkCachePath = null   // set below
            )
            val songId = db.songDao().insert(songEntity)

            // 4. Download + compress artwork (TIDAL provides the URL)
            track.albumArt?.let { artUrl ->
                artworkCacheHelper.downloadAndCompress(songId, artUrl)
            }

            songId
        }
    }
}
```

### SongRepository (no TIDAL — local only)

```kotlin
class SongRepository @Inject constructor(private val db: GrooveDatabase) {

    val allSongs: Flow<List<Song>> = db.songDao().observeAll().map { it.map(SongEntity::toDomain) }

    suspend fun findById(id: Long): Song? = db.songDao().findById(id)?.toDomain()

    /** Called at playback time — pure local DB lookup, zero network */
    suspend fun getPlaybackData(songId: Long): PlaybackData? {
        val song    = db.songDao().findById(songId) ?: return null
        val artwork = db.artworkCacheDao().find(songId)
        val lyrics  = db.lyricsCacheDao().find(songId)
        return PlaybackData(song.toDomain(), artwork?.playerThumbnailPath, lyrics)
    }
}
```

## Issue 6 Fix — Artwork Compression

```kotlin
@Singleton
class ArtworkCacheHelper @Inject constructor(
    @ApplicationContext private val context: Context,
    private val db: GrooveDatabase,
    private val okHttp: OkHttpClient
) {
    private val cacheDir = File(context.cacheDir, "artwork").also { it.mkdirs() }

    suspend fun downloadAndCompress(songId: Long, artUrl: String) = withContext(Dispatchers.IO) {
        val bytes = okHttp.newCall(Request.Builder().url(artUrl).build()).execute()
            .use { it.body?.bytes() } ?: return@withContext

        val original = BitmapFactory.decodeByteArray(bytes, 0, bytes.size) ?: return@withContext

        val listPath   = saveCompressed(original, songId, 128,  "list")
        val playerPath = saveCompressed(original, songId, 512,  "player")
        original.recycle()

        db.artworkCacheDao().upsert(
            ArtworkCacheEntity(songId, listPath, playerPath)
        )
    }

    /** For SAF / MediaStore scanned songs: extract embedded art from file URI */
    suspend fun extractAndCompress(songId: Long, fileUri: Uri) = withContext(Dispatchers.IO) {
        val retriever = MediaMetadataRetriever()
        try {
            retriever.setDataSource(context, fileUri)
            val bytes = retriever.embeddedPicture ?: return@withContext
            val bmp   = BitmapFactory.decodeByteArray(bytes, 0, bytes.size) ?: return@withContext
            val listPath   = saveCompressed(bmp, songId, 128, "list")
            val playerPath = saveCompressed(bmp, songId, 512, "player")
            bmp.recycle()
            db.artworkCacheDao().upsert(ArtworkCacheEntity(songId, listPath, playerPath))
        } finally {
            retriever.release()
        }
    }

    private fun saveCompressed(bmp: Bitmap, songId: Long, sizePx: Int, tag: String): String {
        val scaled = Bitmap.createScaledBitmap(bmp, sizePx, sizePx, true)
        val file   = File(cacheDir, "${songId}_${tag}.jpg")
        FileOutputStream(file).use { scaled.compress(Bitmap.CompressFormat.JPEG, 85, it) }
        scaled.recycle()
        return file.absolutePath
    }
}
```

## Issue 8 Fix — Play History Recording

```kotlin
// Call this from PlayerViewModel whenever a song finishes or is skipped
class PlayHistoryRepository @Inject constructor(private val db: GrooveDatabase) {

    suspend fun recordPlay(
        songId: Long,
        totalPlayedMs: Long,
        durationMs: Long,
        wasSkipped: Boolean
    ) {
        val completion = if (durationMs > 0) totalPlayedMs.toFloat() / durationMs else 0f

        db.playHistoryDao().insert(
            PlayHistoryEntity(
                songId              = songId,
                completionPercentage = completion,
                totalPlayedMs       = totalPlayedMs,
                wasSkipped          = wasSkipped
            )
        )

        // Update aggregate stats
        val existing = db.songStatsDao().find(songId) ?: SongStatsEntity(songId)
        db.songStatsDao().upsert(
            existing.copy(
                playCount      = existing.playCount + 1,
                skipCount      = existing.skipCount + (if (wasSkipped) 1 else 0),
                totalPlayedMs  = existing.totalPlayedMs + totalPlayedMs,
                lastPlayedAt   = System.currentTimeMillis()
            )
        )
    }
}
```
